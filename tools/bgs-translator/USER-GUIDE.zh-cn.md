# bgs-translator 网页 GUI 使用手册

本工具用于把 Bethesda 游戏 MOD 的 `.esp/.esm/.esl` 文本交给 AI 辅助汉化，并导出 xTranslator 可导入的 `.sst` 字典。它不会直接改原始 MOD 文件。最终生成游戏可用翻译文件的步骤仍然在 xTranslator 里完成。

## 适合谁

- 想用 AI 帮忙汉化 Starfield、Skyrim、Fallout 等 BGS 游戏 MOD 的普通玩家。
- 不熟悉插件内部结构、LLM provider、record signature 等术语也可以使用。
- 会打开网页、选择文件、确认 AI 提示词、检查译文即可。

## 启动

```powershell
xtl gui
```

打开后是本机浏览器页面。旧版 Tk 窗口已经删除，浏览器页面是唯一 GUI。

## 基本流程

1. **导入项目**
   - 进入「项目」页。
   - 选择要汉化的 `.esp/.esm/.esl`。
   - 工具会读取可翻译文本，并尽量自动识别游戏。
   - 导入后检查页面上显示的 MOD 文件名、游戏、语言和条目统计是否符合预期。

2. **配置 AI 账号**
   - 进入「AI 设置」页。
   - 新建或选择 provider profile。
   - API key 只保存在本机 `translator/profiles/.env`，不会写进项目文件。
   - 点击「测试连接」确认账号和模型能正常调用。

3. **检查条目**
   - 进入「条目」页。
   - 用「文本类型」「字段」「状态」筛选要处理的内容。
   - 右侧源文/译文面板用于查看完整文本和上下文。
   - 纯占位符、纯数字、只应保持原样的内容不应该交给 AI。

4. **准备术语**
   - 进入「专有名词」页。
   - 玩家专有名词用于固定译法，例如 `Watchtower -> 守望塔`。
   - 禁止翻译用于强制保持原文，例如代码名、变量名、特殊标签。
   - 游戏本体术语会由工具按当前文本自动召回；玩家术语优先级更高。

5. **提交翻译队列**
   - 在「条目」页勾选条目，或使用「疯狂模式」提交当前筛选命中的全部条目。
   - 「每组条数」决定一次 AI 请求里放多少条文本。
   - 如果一次放太多，提示词可能超过模型上下文；这时应减小每组条数或降低术语预算。

6. **预览提示词**
   - 进入「提示词」页。
   - AI 真正翻译前，会先显示本次收到的系统提示词、文本和术语。
   - 检查是否包含正确的游戏背景、MOD 背景、术语和禁止翻译规则。
   - 不确定时只确认当前文本组；确定无误时可以选择本次任务后续自动确认。

7. **查看进度**
   - 进入「进度」页。
   - 查看每个文本组的状态、完成数、失败数和记录。
   - 「请求停止」只保证在安全检查点停止；已经发给 provider 的请求可能仍然计费。
   - 已完成的历史任务可用于审计。没有恢复按钮时，不要假设历史任务能继续运行。

8. **检查和修正**
   - 回到「条目」页筛选「需复查」「未翻译」「锁定」。
   - 需复查通常代表 xTranslator 里的 partial translation，需要人工确认。
   - 锁定通常代表应保持原样，不应再次交给 AI。

9. **导出 SST**
   - 进入「项目」页。
   - 点击「导出 xTranslator 文件」。
   - 点击「打开导出目录」找到 `.sst`。
   - 在 xTranslator 中导入 SST，再由 xTranslator Finalize 生成最终翻译文件。

## 页面说明

- **项目**：导入插件、查看项目状态、record signature 统计、导出 SST。
- **条目**：筛选、查看、编辑、快速翻译、批量提交队列。
- **进度**：查看当前和历史翻译任务，停止任务，审计输出。
- **提示词**：预览 AI 将收到的系统提示词和本组文本。
- **AI 设置**：provider、模型、并发、限速、key、本机保存位置。
- **专有名词**：玩家术语、禁止翻译、游戏本体术语召回结果。
- **记录**：技术日志，排查失败时使用。

## 常见问题

**为什么有些内容没有交给 AI？**

纯数字、纯占位符、脚本变量、`<Alias=...>` 这类内容通常不能翻译。它们应被锁定或跳过，否则会破坏游戏内动态文本。

**为什么 xTranslator 里显示 partial translation？**

这类内容在本工具里应该对应「需复查」。如果 GUI 统计和 xTranslator 不一致，需要先修状态对齐，不要直接发布。

**为什么导出后还要用 xTranslator？**

本工具负责 AI 翻译和导出 SST。xTranslator 负责把 SST 应用到它自己的翻译工程，并 Finalize 成游戏实际读取的字符串文件。

**可以不预览提示词直接翻译吗？**

可以，但只建议熟悉当前项目和模型时使用。普通玩家建议保留预览，避免一次性把错误规则发给 AI。

**费用显示能相信吗？**

不同 provider 和中转站计价规则不同。本工具不应替代 provider 官网账单。大批量任务前请自行确认余额和模型价格。

## 给 AI Agent 的协作建议

如果你让 AI Agent 帮忙跑完整流程，推荐这样要求：

1. 先调查 MOD 页面、readme 和条目样本，写详细 game context 和 mod context。
2. 再生成 system prompt，交给你预览。
3. 你确认后再开始批量翻译。
4. 翻译完成后让 Agent 导出 SST，并说明哪些条目还需复查。

不要让 Agent 直接修改原始插件文件。
